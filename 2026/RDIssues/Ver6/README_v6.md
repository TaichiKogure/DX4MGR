# DX4MGR Ver6: 差し戻し負荷・プロセス反映モデル

## 概要
Ver6では、実務フィードバックに基づき、差し戻し（Rework）が発生した際の「負荷の重み」をモデル化しました。
「差し戻されると追加の小実験が増える」という現実の挙動を `rework_load_factor` パラメータで表現し、プロセス停滞のリスクをより正確に可視化します。

## 主な新機能
- **rework_load_factor**: 差し戻し時のサービス時間増加係数。
- **差し戻し回数分布の可視化**: 案件ごとの差し戻し状況を箱ひげ図で表示。
- **DR会議/試作検証スパンへの対応**: 長期シミュレーション（365日〜）と低頻度ゲートの評価に対応。

## ファイル構成
- `simulator_v6.py`: 負荷増大ロジックを含む離散イベントシミュレータ。
- `analyzer_v6.py`: 統計解析と検証ゲート（Quality Gates）の判定。
- `visualizer_v6.py`: CI、分布、CCDF、差し戻し回数などの可視化。
- `run_v6.py`: シミュレーション実行パイプライン。
- `scenarios.csv`: シミュレーション条件設定ファイル。

## 実行方法
```bash
python3 2026/RDIssues/Ver6/run_v6.py
```

## 可視化結果の見方
1. `v6_step1_doe_analysis.png`: どのパラメータがスループットに最も影響を与えるか（感度分析）。
2. `v6_step2_gate_*.png`: 各シナリオが目標基準（スループット、待ち時間、安定性、差し戻し回数）を満たしているか。
3. `v6_step3_compare_reworks.png`: シナリオ間で差し戻しの発生頻度・重みがどう違うか。
4. `v6_step3_comparison_ci.png`: 統計的有意差を含む改善効果の比較。
