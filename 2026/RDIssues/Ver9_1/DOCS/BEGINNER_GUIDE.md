# DX4MGR Ver7 初心者ガイド

DX4MGR（Digital Transformation for Manager） Ver7へようこそ。
このツールは、研究開発や設計業務のような「不確実性が高く、差し戻し（Rework）が発生しやすい業務」をシミュレートし、最適な改善策を統計的に見つけるためのツールです。

## 3分でわかる解析の流れ

### 1. 問いを立てる
まずは何を改善したいか決めましょう。
- 「承認者のスキルを上げたら、プロジェクトの期間は短縮されるか？」
- 「差し戻しが起きた時の手直し負担（重み）が増えると、どれくらい遅延するか？」

### 2. シナリオを設定する (`scenarios.csv`)
`scenarios.csv` を編集して、比較したい条件を書き込みます。
- **01_Baseline**: 今の状況（基準）。
- **02_Challenge**: 改善案や、逆にリスクが起きた状況。
各パラメータの意味は `PARAMETER_GUIDE.md` を参照してください。

### 3. シミュレーションを実行する
ターミナルを開き、以下のコマンドを入力します。
```bash
python3 2026/RDIssues/Ver7/run_v7.py
```

### 4. 結果を読み解く
`output/` フォルダに生成されるグラフを確認します。
- **合格か不合格か？** (`v7_step2_gate_...png`)
  - PASS（緑）が出ていれば、そのシナリオは目標基準（スループットや待ち時間）をクリアしています。
- **本当に効果があるか？** (`v7_step3_comparison_ci.png`)
  - 棒グラフの「ヒゲ」が重なっていなければ、その改善は統計的に意味があります。

## 困ったときは
- **「ModuleNotFound...」が出る**: 必要なライブラリ（pandas, numpy, matplotlib）がインストールされているか確認してください。
- **「Error: scenarios.csv not found」**: Ver7では修正済みですが、もし出た場合はスクリプトのフルパスを使って実行してください。
