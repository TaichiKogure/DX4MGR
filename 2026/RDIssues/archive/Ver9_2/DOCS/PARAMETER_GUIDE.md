# DX4MGR Ver7 パラメータ解説ガイド

シミュレーションで使用する主要パラメータの意味と設定のヒントを解説します。

## 1. 案件の投入と処理 (Flow)
- **days**: シミュレーションを行う期間（日）。長期的な傾向を見るには 180〜365 日を推奨します。
- **arrival_vp_per_day**: 1日あたりの平均案件投入量（VP: Value Point）。
- **service_vp_per_day**: 1つのリソース（承認者）が1日に処理できる平均量。
- **approvers**: 承認者（または処理リソース）の数。

## 2. 差し戻しと負荷 (Rework & Load)
- **p_rework**: 差し戻しが発生する確率 (0.0 〜 1.0)。
- **rework_load_factor**: **(重要)** 差し戻しが発生した際に、追加で発生する作業負荷の倍率。
  - 1.0 = 最初と同じだけの時間がかかる。
  - 0.5 = 半分の時間で修正できる。
  - 2.0 = 修正のために最初の2倍の時間がかかる（実務でよくある泥沼状態）。

## 3. 能力と環境 (Skill & Environment)
- **skill_factor**: 全体的な作業効率。1.2に設定すると20%効率が上がります。
- **dependency_level**: 他部署や外部との調整の多さ (0.0 〜 1.0)。高いほど待ち時間が増えます。
- **skill_at_dependency_resolution**: 調整をスムーズに行う能力。これが高いと dependency_level の悪影響を軽減できます。

## 4. シミュレーション精度
- **n_trials**: シミュレーションを何回繰り返すか。
  - 30回: ざっくりした傾向を見る。
  - 100回: 標準的な解析。
  - 300回: 厳密な統計的有意差を確認したい場合。
