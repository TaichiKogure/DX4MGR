# R&D Flow 健全性診断 マニュアル (Ver2: パラメータ補強版)

## 1. 導入背景と目的
R&D組織の生産性は、単なる「個人の作業スピード」では測れません。
特に電池・電子部品開発のような多拠点・多部門が絡む現場では、**「意思決定の滞留（承認待ち）」**と**「部門間の依存関係（調整コスト）」**が最大のボトルネックとなります。

Ver2モデルでは、以下の2点を強化しました：
- **スキルの定量化**: 意思決定の質とスピード、および調整能力。
- **依存性の構造化**: 組織の壁がどれだけスループットを削っているかの可視化。

---

## 2. パラメータ解説 (マニュアル)

### ① スキル因子 (Skill Factor)
- **定義**: 承認者（意思決定者）の専門知識や判断の速さ。
- **設定の目安**:
  - `1.0`: 標準。
  - `1.5`: 熟練のエキスパート。資料の行間を読み、即座に的確な判断ができる。
  - `0.7`: 経験の浅い担当。判断に迷いが多く、確認作業に時間がかかる。

### ② 依存関係解消能力 (Dependency Resolution Skill)
- **定義**: 部門を跨ぐ調整や、複雑な依存関係を解きほぐすソフトスキル。
- **設定の目安**:
  - `0.0`: 調整は成り行き任せ。
  - `1.0〜2.0`: 高い調整能力。事前に他部門と合意形成（根回し）を行い、承認ゲートでの「想定外」を減らせる。

### ③ 依存レベル (Dependency Level)
- **定義**: プロセス完遂に必要な他部門・他リソースへの依存度。
- **設定の目安**:
  - `0.1`: チーム内でほぼ完結する（自律型）。
  - `0.5`: 試作・測定・評価などで他部門の承認やリソース確保が必須（分業型）。

### ④ 調整ペナルティ (Coordination Penalty)
- **定義**: 組織の官僚化やルールの複雑さ。
- **設定の目安**:
  - `1.0`: 標準。
  - `2.0`: 会議体が多く、手続きが重い。

## 3. 考察のTIPS (診断の読み解き方)

### TIPS 1: 「忙しいのに成果が出ない」の正体
稼働率が1.0に近いのにスループットが低い場合、`dependency_level` を疑ってください。
個人のスキルを上げても、組織の壁（依存関係）が高いままでは、実効キャパシティは摩擦熱のように調整コストで消費されてしまいます。

### TIPS 2: 承認者を増やすか、スキルを上げるか
ボトルネックが「承認」にある場合、安易に人数を増やす（`approvers`を増やす）前に、`skill_factor`（判断の標準化・テンプレ化）や `skill_at_dependency_resolution`（事前レビュー）を上げる方が、投資対効果（ROI）が高いことが多いです。

### TIPS 3: 「再作業」と「依存」の悪魔の双子
依存関係が多い組織では、差し戻し（`p_rework`）が発生した際の影響が倍増します。
1回の差し戻しが、再び「他部門との調整待ち」を引き起こすためです。シミュレーションで `p_rework` と `dependency_level` を同時に上げた際のスループット崩壊を確認してください。

## 4. 理論的背景
本モデルは、以下の組織・チーム理論をベースに設計されています。

- **IPO (Input-Process-Output) モデル**:
  - `Input` (スキル、リソース) が `Process` (相互作用、調整) を経て `Output` に変換される構造をモデル化しています。
- **Job Characteristic Theory (職務特性理論)**:
  - スキルの多様性や自律性（依存の少なさ）が、どのようにパフォーマンスと結びつくかをパラメータに反映しています。
- **R&D Capability / Absorptive Capacity (吸収能力)**:
  - 外部や他部門の知識を吸収し、調整する能力が、組織の実効スループットを決定付ける重要な因子であることを示しています。

## 5. 改善アクションへの繋げ方

| 兆候 | 推奨されるアクション |
| :--- | :--- |
| **待ち時間が長い** | 承認ゲートの分散、または判断基準の明文化（スキル因子の底上げ） |
| **再作業が多い** | Definition of Ready (DoR) の導入、事前チェックリストの整備 |
| **稼働率が異常に高い** | 依存関係の削減（マイクロサービス的組織化）、または調整担当の設置 |

*このシミュレータは、管理のためではなく、組織の「認知補助装置」として活用してください。*
