# DX4MGR Ver3: 可視化・説明力強化モデル

Ver3では、シミュレーション結果を単なる数値ではなく、**「上層部の意思決定を促す」**ための強力な可視化フローを実装しました。

## 1. 構成プログラム
- `simulator_v3.py`: 詳細ログ収集（待ち時間、WIP、複数試行）に対応したコアエンジン。
- `visualizer_v3.py`: 各種グラフ（ウォーターフォール、トルネード、バイオリン、CCDF等）の描画モジュール。
- `run_v3.py`: ステップバイステップで可視化を実行し、`output/` ディレクトリに保存するメインスクリプト。

## 2. 4つのステップと可視化の狙い

### Step 1: 実効処理能力の内訳 (Waterfall)
- **狙い**: 「なぜ忙しいのに成果が出ないのか」を構造的に示す。
- **グラフ**: 寄与分解ウォーターフォール。
- **説明のコツ**: 「スキルで+20%向上させたが、組織間の依存と調整コストで-30%削られている」といった、掛算構造を足し算のように見せることで、ボトルネックを明確にします。

### Step 2: 改善の優先順位 (Tornado)
- **狙い**: 「次に何を打つべきか」をデータで優先順位付けする。
- **グラフ**: 感度分析トルネードチャート。
- **説明のコツ**: 「承認者を増やすより、手戻り（再作業）を10%減らす方がスループットへの影響が2倍大きい」といった比較が直感的に伝わります。

### Step 3: 詰まりの怖さ (Violin / WIP / CCDF)
- **狙い**: 「平均値」の嘘を暴き、現場の崩壊リスクを伝える。
- **グラフ**: P90/P95強調分布、WIP時系列、CCDF（サバイバル曲線）。
- **説明のコツ**: 
  - **P90待ち時間**: 「平均は5日ですが、10件に1件は14日を超えます。この“尾っぽ”が現場の炎上と信頼低下の原因です」
  - **CCDF**: 「納期を10日以内と約束した場合、現状では30%の確率で事故ります」
  - **WIP**: 「稼働率が1.0に近づくと、滞留数が線形ではなく爆発的に増える」破綻の瞬間を見せます。

### Step 4: 組織構造の壁 (Network)
- **狙い**: 依存関係を「絵」にして、組織改編や境界再設計の必要性を説く。
- **グラフ**: 依存ネットワーク図。
- **説明のコツ**: エッジの太さでコミュニケーション・待ちの負荷を可視化し、特定部門への集中をあぶり出します。

### Step 5: エグゼクティブ・ダッシュボード
- **狙い**: 経営層が1分で状況を把握し、投資判断（改善案へのGo）を出せるようにする。
- **表示**: P50/P90納期比較、および遅延リスク信号。

## 3. 実行方法
以下のコマンドで、全ての可視化サンプルが生成されます。
```bash
python3 2026/RDIssues/Ver3/run_v3.py
```
結果は `2026/RDIssues/Ver3/output/` に保存されます。
